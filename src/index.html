Logo

Divisor

Showcase Divisor

                    <dsw-divisor appearance="divisor" ></dsw-divisor>

Propriedades

appearance: Você poderá subscrever as propriedades desta divisória (cores, etc). Para subscrever as propriedades, basta criar uma classe para subscrever as regras css. Caso contrário segue as regras padrões abaixo: 

              .divisor {

                  border-radius: var(--border-radius-none);

                  border: var(--border-widths-small);

                  border-color: var(--dark-color-base-02);

                  min-width: 80px;

              }

Input(s)

Showcase dos Input(s)

                    <dsw-input inputId="input_id" required="true" type="text" label="Label do Input"></dsw-input>
                  

                    <dsw-input inputId="input_id" required="true" type="text" label="Label do Input" icon="assets/img/icon.svg"></dsw-input>
                  

                    <dsw-input inputId="input_id" required="true" type="text" label="Label do Input" icon="assets/img/icon.svg" text-error="Texto de exemplo de erro"></dsw-input>
                  

                    <dsw-input inputId="input_id" required="true" type="text" label="Label do Input" icon="assets/img/icon.svg" text-error="Texto de exemplo de erro"></dsw-input>

Propriedades

O componente Input possui as seguintes propriedades:

  • label: Atributo usado para inserír o texto da label do Input.
  • inputId: Atributo responsável pelo ID do Input e também da label deste Input.
  • type: Atributo auto-definido e usado para definir o tipo de Input (submit, button, reset, etc). Veja mais em : https://www.w3schools.com/tags/tag_input.asp
  • icon: Atributo usado para indicar o path/ caminho da imagem desejada para ser usada como icone do input.
  • required: Atributo auto-definido e usado para indicar se o input é ou não obrigatório/ requerido.
  • textError: Atributo usado para mostrar o texto de Erro desejado.

OBS: Os Atributos "icon" e também "textError" são opcionais e ao serem declados alteram visualmente o componente. Adicionando o ícone como no exemplo acima e no caso do textError adiciona as cores e mensagem de erro abaixo do Input.
